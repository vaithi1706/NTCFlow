1. Introduction & Overview

DKFlow is an AI-powered project management platform designed for teams of all sizes. It combines traditional project management with 18+ AI features to help teams plan, track, and deliver work more efficiently.

✦ Key Capabilities
  • Kanban Board, List, Table, Calendar views
  • Sprint planning & Agile workflows
  • Epics, Roadmap, Dependencies
  • Document version control
  • Goals & OKRs tracking
  • Time tracking & reports
  • AI task generation & estimation
  • AI meeting summarization
  • AI-powered search (Intelligence Engine)
  • Role-based access control (10 roles)
  • Team workload management
  • Custom forms & approvals

System Requirements

  • Browser: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
  • Internet: Stable connection required
  • Access: Visit https://dkflow.in

2. Registration & Login

2.1 Creating an Account

  1. Navigate to dkflow.in and click "Get Started"
  2. Enter your Full Name, Email, and Password (min 8 characters)
  3. Select your role (Project Manager, Developer, Designer, etc.)
  4. Click "Create Account"
  5. Verify your email address via the confirmation link

2.2 Logging In

  1. Go to dkflow.in/login
  2. Enter your registered email and password
  3. Click "Sign In" — you'll be redirected to your Home dashboard
💡 Tip: Use the "Remember me" option for faster login on trusted devices.

3. Workspace Setup

A Workspace is the top-level container for your organization. All projects, teams, and members live inside a workspace.

3.1 Creating a Workspace

  1. After login, click "Create Workspace"
  2. Enter a Workspace Name (e.g., "Acme Corp")
  3. Set a URL slug (e.g., acme-corp)
  4. Choose workspace visibility: Private or Public
  5. Invite team members by email (optional — can be done later)

3.2 Workspace Structure

→

Subtasks

Each workspace can contain unlimited projects. Each project contains tasks organized in sprints, epics, or backlogs.

4. Projects

4.1 Creating a Project

  1. Click the "+" button in the sidebar or go to Portfolio → New Project
  2. Enter Project Name, Key (e.g., DKF), and optional description
  3. Choose a template: Blank, Scrum, Kanban, or Bug Tracking
  4. Set Lead, Start Date, and Target Date
  5. Click "Create Project"

4.2 Project Dashboard

Every project has a dedicated dashboard showing:

  • Task Summary: Total, in progress, completed, overdue
  • Sprint Progress: Current sprint burndown
  • Team Activity: Recent changes and comments
  • Priority Distribution: Visual breakdown of task priorities

5. Task Management

5.1 Creating a Task

  1. Open any project view (Board, List, etc.)
  2. Click "+ New Task" or press N keyboard shortcut
  3. Enter the Task Title
  4. Set optional fields: Assignee, Priority, Due Date, Labels
  5. Press Enter or click "Create"

5.2 Task Properties

PropertyOptionsDescription
StatusTodo, In Progress, Review, DoneCurrent state of the task
PriorityNone, Low, Medium, High, UrgentImportance level
AssigneeAny project memberWho is responsible
Due DateAny dateDeadline
LabelsCustom colored tagsCategorization
SprintActive/future sprintsSprint assignment
EpicProject epicsParent epic
Story Points1–21 (Fibonacci)Effort estimation
Time EstimateHoursPlanned hours

5.3 Task Detail View

Click any task to open its detail panel. Features include:

  • Rich text description with markdown support
  • Comments with @mentions
  • Subtasks — break down work further
  • Activity log — full change history
  • Attachments — upload files
  • Time logs — track time spent
  • Dependencies — link blocking/blocked-by tasks
💡 AI Feature: Click "AI Suggest" to get AI-generated subtasks, estimates, and descriptions based on the task title.

6. Views — Board, List, Table, Calendar, Timeline

6.1 Board View (Kanban)

The default view. Tasks appear as cards in columns representing statuses.

  • Drag & drop cards between columns to change status
  • Filter by assignee, priority, label, or sprint
  • Group by status, assignee, priority, or label
  • Quick edit — click card to open details

6.2 List View

Simple, scannable list of all tasks. Best for quick triaging.

6.3 Table View

Spreadsheet-style view with inline editing. Click any cell to edit directly. Supports sorting and column customization.

6.4 Calendar View

Tasks displayed on a monthly/weekly calendar based on due dates. Drag tasks to reschedule.

6.5 Timeline View (Gantt)

Horizontal bars showing task duration and dependencies. Perfect for project planning and identifying critical paths.

7. Sprints & Agile

7.1 Creating a Sprint

  1. Go to your project → Sprints tab
  2. Click "New Sprint"
  3. Set Sprint Name, Start Date, End Date (typically 1–2 weeks)
  4. Click "Create Sprint"

7.2 Sprint Planning

  1. Open the Backlog view to see unassigned tasks
  2. Drag tasks from Backlog into the active Sprint
  3. Assign story points to estimate effort
  4. Review sprint capacity vs. planned points
  5. Click "Start Sprint" when ready

7.3 Sprint Charts

Each sprint includes:

  • Burndown Chart: Track remaining work over time
  • Velocity Chart: Compare story points completed across sprints
  • Sprint Report: Summary of completed, incomplete, and added tasks

8. Epics & Roadmap

8.1 Epics

Epics are large features that span multiple sprints. Each epic contains multiple tasks.

  1. Go to project → Epics tab
  2. Click "New Epic"
  3. Set name, description, color, start/end dates
  4. Assign tasks to the epic from the task detail view

8.2 Roadmap View

The Roadmap displays epics on a horizontal timeline. Use it to:

  • Visualize the product plan across quarters
  • Drag epics to adjust timing
  • Share with stakeholders for alignment

9. Dependencies

Link tasks that depend on each other to prevent scheduling conflicts.

Dependency Types

TypeMeaning
BlocksThis task must finish before the linked task can start
Blocked byThis task cannot start until the linked task finishes
Related toTasks are related but not blocking

Setting Dependencies

  1. Open a task detail view
  2. Scroll to "Dependencies" section
  3. Click "Add dependency"
  4. Search for the related task and select the dependency type

The Dependencies tab shows a visual graph of all task relationships in the project.

10. Teams & Members

10.1 Inviting Members

  1. Go to Members in the sidebar
  2. Click "Invite Members"
  3. Enter email addresses (comma-separated for multiple)
  4. Select a role (Admin, Member, Guest)
  5. Click "Send Invitations"

10.2 Creating Teams

  1. Go to Teams in the sidebar
  2. Click "Create Team"
  3. Name the team (e.g., "Backend Team", "Design")
  4. Add members to the team
  5. Assign the team to projects

12. Approvals

Request and manage approvals for tasks, documents, or decisions.

  1. Go to Approvals in the sidebar
  2. Click "New Approval"
  3. Enter the subject and description
  4. Select approver(s) from workspace members
  5. Attach relevant tasks or documents
  6. Submit — approvers receive notification

13. Notifications

Stay updated with real-time notifications for:

  • Task assignments and status changes
  • Comments and @mentions
  • Sprint starts/completions
  • Approval requests
  • Due date reminders

Notification Settings

Configure per-channel notifications in Workspace Settings → Integrations:

  • In-app: Bell icon in top navigation (always on)
  • Email: Toggle email digests (instant, daily, weekly)
  • Slack/Teams: Add webhook URL for channel notifications

14. Documents Module

Full document management with version control, categories, and AI features.

14.1 Uploading Documents

  1. Go to project → Documents tab
  2. Click "Upload Document" or drag & drop files
  3. Set title, category, and tags
  4. Optionally link to tasks
  5. Click "Upload" (max 50MB per file)

14.2 Version Control

Upload new versions of existing documents. DKFlow tracks all versions with:

  • Version number (auto-incremented)
  • Upload date and uploader
  • Version notes/changelog
  • Ability to download any previous version

14.3 Organization

  • Categories: Create hierarchical categories/sub-categories
  • Tags: Multi-tag system for flexible filtering
  • Search: Full-text search across all documents
  • Task linking: Bidirectional links between documents and tasks
💡 AI Feature: Auto-tagging, AI-generated summaries, and semantic search coming in v2.

16. Meetings & Transcripts

16.1 Adding a Meeting

  1. Go to Meetings in the sidebar
  2. Click "Add Manually" (or connect Microsoft Teams for auto-sync)
  3. Enter meeting subject and attendees
  4. Paste the meeting transcript or notes
  5. Click "Add Meeting"

16.2 AI Processing

  1. Open the meeting and click "AI Process"
  2. AI generates: Summary, Key Decisions, Action Items
  3. Review the extracted action items (each has title, assignee, priority, due date)
  4. Click "Create Tasks" to automatically create DKFlow tasks

16.3 Microsoft Teams Integration

Optionally connect your Microsoft Teams account to auto-sync meetings and transcripts. Requires organizational admin consent for transcript access.

17. Goals & OKRs

17.1 Creating a Goal

  1. Go to Goals in the sidebar
  2. Click "New Goal"
  3. Enter Goal Title and Description
  4. Set Key Results (measurable outcomes)
  5. Link related projects or tasks
  6. Set owner and time period (quarterly/annual)

17.2 Tracking Progress

Goals automatically track progress based on linked tasks and key results. Update key result values to reflect current progress. The goal percentage is auto-calculated.

18. Workload & Resource Planning

18.1 Workload View

Go to Workload in the sidebar to see each team member's task load displayed as a horizontal capacity bar chart.

  • Green: Under capacity — available for more work
  • Yellow: Near capacity — balanced
  • Red: Over capacity — redistribute work

18.2 Resource Planning

The Resource Planning view shows team allocation across projects and time periods. Use it to:

  • Identify over/under-utilized team members
  • Plan resource allocation for upcoming sprints
  • Balance workload across the team

20. Time Tracking

20.1 Logging Time

  1. Open a task detail view
  2. Click "Log Time" or the timer icon
  3. Enter hours and optional description
  4. Click "Save"

20.2 Time Reports

Go to project → Time Report tab to see:

  • Total hours logged per task and member
  • Estimated vs actual hours comparison
  • Weekly/monthly breakdown
  • Export to CSV

24. Admin Panel

The standalone Admin Panel (port 8443) provides super-admin controls:

  • Users: View all registered users, activate/deactivate accounts
  • Workspaces: Manage all workspaces, view usage statistics
  • Licenses: Generate, activate, and manage Pro license keys
  • System Health: API status, database connections, uptime
⚠️ Admin Panel is separate from the main DKFlow app. Access it at https://dkflow.in:8443 with admin credentials.
